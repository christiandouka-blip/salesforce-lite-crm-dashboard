# Salesforce CRM Clone Rules

## Mission

Build a high-fidelity Salesforce CRM clone using only the provided screenshots, PDFs, HTML files, and explicit user instructions.

The objective is replication.

Not interpretation.

Not redesign.

Not inspiration.

---

## Rule 1

Do not design.

Clone.

---

## Rule 2

Do not improvise.

Replicate.

---

## Rule 3

Do not assume missing functionality.

If functionality is unclear:

Stop.

Ask.

---

## Rule 4

Read all screenshots before writing code.

---

## Rule 5

Read all PDFs before writing code.

---

## Rule 6

Read all HTML files before writing code.

---

## Rule 7

Analyze every reference asset.

Produce a report.

Wait for approval.

Only then begin implementation.

---

## Rule 8

Do not create screens that do not exist in references unless explicitly approved.

---

## Rule 9

Do not remove screens that exist in references.

---

## Rule 10

No placeholder layouts.

---

## Rule 11

No lorem ipsum.

---

## Rule 12

No fake navigation.

---

## Rule 13

No dead buttons.

---

## Rule 14

No dead links.

---

## Rule 15

Every clickable element must have a destination.

---

## Rule 16

Every navigation path must work.

---

## Rule 17

Every screen must connect correctly to the next screen.

---

## Rule 18

Every form must have working validation.

---

## Rule 19

Every modal must open and close correctly.

---

## Rule 20

Every dropdown must function.

---

## Rule 21

Every table must behave correctly.

---

## Rule 22

Every sidebar item must navigate correctly.

---

## Rule 23

Every top navigation item must navigate correctly.

---

## Rule 24

Replicate visual details exactly:

* spacing
* margins
* padding
* typography
* font sizes
* colors
* borders
* shadows
* alignment
* icon placement
* component hierarchy

---

## Rule 25

Do not approximate layouts.

Match reference layouts.

---

## Rule 26

Do not simplify layouts.

---

## Rule 27

Do not replace components with alternatives.

---

## Rule 28

Do not change workflows.

---

## Rule 29

Do not rename screens.

---

## Rule 30

Do not rename navigation items.

---

## Rule 31

Before building any screen:

Provide:

* reference files used
* screen analysis
* component inventory
* functionality inventory
* navigation inventory

Wait for approval.

---

## Rule 32

Build one screen at a time.

---

## Rule 33

Do not build the next screen until the current screen is approved.

---

## Rule 34

After building a screen:

Provide:

* completed features
* remaining features
* known gaps

Wait for approval.

---

## Rule 35

Use reusable components whenever possible.

---

## Rule 36

Keep code modular and maintainable.

---

## Rule 37

Use semantic HTML.

---

## Rule 38

Keep CSS organized.

---

## Rule 39

Keep JavaScript organized.

---

## Rule 40

Before marking any screen complete:

Verify:

* visual match
* navigation match
* interaction match
* responsive behavior
* accessibility basics

---

## Rule 41

Dashboard behavior must match references.

---

## Rule 42

Login behavior must match references.

---

## Rule 43

The user is the final authority.

If the user requests a change:

Pause.

Re-evaluate.

Proceed only after approval.

---

## Rule 44

Never claim a feature is complete without verification.

---

## Rule 45

When uncertain:

Ask.

Never assume.
